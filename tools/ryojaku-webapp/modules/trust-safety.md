# 🛡️ 信任與安全 (Trust & Safety) — 反托機制

> **狀態**：foundation 風險控管層，與金流紅線同層 🔶（Claude 提案，結構化給 Codex 實作/測試）
> 一句話：**防「麻將館員工假扮散客導客(托)」——讓誠實揭露比偷藏划算、館家為員工問責;偵測只導向人工覆核，絕不自動定罪。**

反托是 **foundation 層信任邊界**：帳本可信度、排名可信度、RPG 成長公平性、LINE 檢舉/審核入口全建在它上面(接 §2.11)。

---

## 1. 威脅模型

麻將館員工/樁腳**假扮個人玩家**在平台找牌咖，把人**導向自家商業場**消費。用戶到現場才發現「牌搭是店家的人」→ 直擊護城河(信任)。

## 2. 核心原則(治本 > 抓人)

1. **給館家漂亮正門**：認證徽章 / 置頂 / 優先搜尋 / 折價券(business §2.5/2.6)→ **掛牌店家代表**生意更好 → 偷藏誘因下降。
2. **把責任壓館家**:平台在賣館家 B2B = 有籌碼。認證條款寫明「旗下員工冒充個人玩家導客 → **撤認證 + 停曝光 + 退費追究**」→ 逼館家自管員工。
3. **揭露 = 升級不是懲罰**:商業招募走認證身分、名片掛「🏪 XX館 店家代表」→ 有發券/曝光特權。**說謊才是違規,不是「有商業身分」**。

## 3. 偵測訊號 → 風險分(只讀平台既有資料，不新增監控)

| 訊號 | 指向托的理由 | 資料源 |
|---|---|---|
| 地點集中度 | 約局/打卡幾乎全在同一店(散客會跑不同場/在家打) | 打卡、場域 tag |
| 關係圖單向 | 永遠對陌生人開局、零回頭牌搭、高換咖率+全同店 | 對局關係圖 |
| 導向行為 | 每次都把地點導向某商業場 | 團局 venue |
| 帳號簇 | 多「玩家」帳號共裝置/IP/金流、全 funnel 同店 | 反分身(onboarding) |
| 非牌局時段在店 | geofence 顯示像員工長駐 | 打卡時序 |
| 評價/檢舉語意 | 出現「原來是店家的人/托」 | reputation 檢舉 |

→ 加權成**風險分**;敏感訊號(裝置/IP 簇)**僅用於累犯/高風險覆核**，不進日常評分。

## 4. 處置流程(risk → 人工，非 risk → 罰)

```
風險分 ≥ 門檻
  → 私下詢問:「你與此場館是否有商業/受雇關係?」(給乾淨揭露出口)
    → 誠實揭露 → 轉為「店家代表」身分(掛標、不罰)
    → 隱瞞/謊報，事後查證屬實 → 認定違規
         → 個人:名片標示 → 媒合降權 → 累犯封停
         → 關聯館家(需已建立關聯):認證扣分 / 撤認證 / 停曝光
```

## 5. 🚫 不可逾越的紅線 = 不變式(給 Codex 寫測試)

1. **不自動定罪**:風險分 ≥ 門檻**永不**自動施加任何懲罰/公開標籤 → **必經人工覆核**。風險分只能觸發「詢問+覆核」。
2. **違規的充要條件 = 說謊**:唯有「被要求揭露時**隱瞞/謊報**商業關係」構成違規。單純「常在某店打 / 高勝率 / 地點集中」**永不**單獨構成違規。
3. **誠實揭露豁免**:主動或被問即揭露的店家代表**不受反托懲罰**,只降級為「需掛店家標」。
4. **館家連坐需已建立關聯**:對館家的處分只在**已申報/已查證關聯**下成立，**不得**僅憑風險分推定連坐。
5. **最小監控**:偵測只讀既有平台資料，不新增額外蒐集;裝置/IP 簇僅限高風險覆核。
6. **可申訴可回復**:誤判可申訴，處分隨後續正常行為衰減回復(對齊 reputation 負評可回升)。

## 6. 接點(用既有模組，不另造)

- **reputation**:檢舉/差評新增分類「**未揭露店家身分(托)**」→ 餵風險分 + 課責。
- **parlor**:認證館家入駐**申報旗下代表/員工名單** → 建立「館家↔人」關聯(不變式#4 的依據)。
- **onboarding**:帳號簇偵測重用**反分身**工具。
- **ladder / RPG**:確認違規者的可信對局**排除或標示**，不污染排名/成長公平性。
