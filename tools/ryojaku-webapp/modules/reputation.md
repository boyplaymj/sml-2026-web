# ⭐ 信譽評價 (Reputation)

> **狀態**：核心模型已由 gameboy 拍板（**Steam 式二元推薦**）🔶 其餘為 Claude 建議預設，待微調
> 一句話：**打完一場可信對局，同桌互評「還想不想再一起打」，理由分維度可篩選查看，累積成名片上的信任招牌。**

護城河 = 信任 → 這是全 app **最重要的社會系統**。建立在現有 `rate-user` / `rate-game` / `submit-rating` + `reputation{ goodRate, goodCount, badCount, tags[] }` **之上升級，不重造**。

---

## 1. 核心機制：Steam 式二元推薦 ✅（拍板）

- **裁決二元、可跳過**：`👍 想再一起打` / `👎 不想再約`。不用星等（不膨脹、不用喬 1–5）。直接回答名片鐵律的問題「**我該不該跟這人打牌**」。
- **理由對稱**：👍 與 👎 **都能寫理由** = 選**分類標籤**（可多選）+ 可選一句話。
- **理由分類 = 篩選維度**：讀者可**只看自己想看的分類**（例:「只看關於『結算』的評價」→ 好壞都列出）。維度對映 §2.1 四大痛點：

  | 分類 | 對映痛點 | 備註 |
  |---|---|---|
  | ⏰ 準時・出席 | 放鴿子(頭號痛點) | **半自動**:no-show 由主揪標記為佐證，較客觀 |
  | 🀄 牌品・情緒 | 牌品差 | 態度、輸不起、悔牌、拖手速 |
  | 💵 結算爽快 | 金錢糾紛 | 🔒 措辭一律「結算爽快/認帳」，**不用「還錢/欠款」**(金流紅線 §2.8) |
  | 🤝 相處・氣氛 | — | 好聊、帶氣氛 |

- **聚合顯示**：沿用**好評率 `goodRate`** 當頭條 + **質性招牌 tier**(仿 Steam「極度好評/褒貶不一」)，把冷冰冰的百分比變成名片上的招牌(呼應 §1「把戰績經營成看得到的招牌」)。
  - 示意階梯：`牌搭首選` → `風評極佳` → `風評不錯` → `褒貶不一` → `風評普通` → `慎約`（門檻=好評率 × 場次基數）。
  - **新手保護**：未滿 N 場(預設 5)只顯示「🌱 新玩家」中性徽章，**不曝光小樣本難看的百分比**。

## 2. 誰評誰・何時・防濫用

- **評權地基(最強防濫用)**：**只有同桌「可信對局」的人才生評權**(接 §3.4 扇出)。沒同桌過→沒評價權 → 擋掉路人刷評、前任來洗、沒打過牌亂評。
- **評價窗口**：對局確認成立後開 **72h** 評價窗，過期封存。
- **對稱盲評**（防報復骨幹）：雙方互評**都送出、或窗口關閉後才同時揭曉**(仿 Airbnb)→ 根除「你給我差評我就報復」。
- **差評需選 ≥1 原因分類**(不能無理由差評)；**被評者有回應權**(可附一句公開回覆，但不能刪對方)。
- **負評非死刑**：同一對人反覆互評**不疊加**(取最近 N 次)；評價**時間衰減** → 後續好表現可覆蓋回升，給改過的路。
- **爭議中的場**：該場評價**一起擱置**(與 §3.4 對齊)。
- **安全 / 嚴重事件**：走**檢舉 → 主揪/管理覆核**，不靠評分解決。檢舉分類含「**未揭露店家身分(托)**」→ 餵 [信任與安全](trust-safety) 風險分,不自動定罪。
- **helpfulness「這則有幫助」排序**：Steam 有，但 500 人小池先**列為 v2 可選**，不急。

## 3. 名片顯示(名片鐵律)

- 頭條：**好評率 + 招牌 tier + 場次基數**。
- **理由標籤雲** + **可篩選的評價列表**(依 👍/👎、依分類、依時間)。
- **準時率**可獨立顯示(半自動、較客觀，且是頭號痛點)。

---

## 4. 資料模型 🔶(沿用並擴充現有 `Rating`)

```
Rating         { id, gameId(必), fromUserId, targetUserId,
                 verdict(up/down), reasonTags[], comment,
                 createdAt, revealedAt, hostFlaggedNoShow? }   // gameId=評權憑證
ReputationSummary(掛 User.reputation)
               { goodRate, upCount, downCount, games,
                 tagCounts{tag:n}, tier, punctualityRate }
ReasonTag 字典  { key, group(4分類), label, polarity }          // 後台可增減
```
評權校驗：`gameId` 必須是「fromUser 與 targetUser 同桌、且已確認成立」的可信對局。

## 5. API 🔶(沿用 App 命名慣例)
`POST /submit-rating {gameId,targetUserId,verdict,reasonTags[],comment}` · `GET /reviews/:userId?verdict=&tag=`(篩選) · `GET /rate-game/:id` · 內部：對局確認成立→開窗；窗口關→**同時揭曉 + 重算聚合**。

## 6. 後台面板(v2)
理由分類字典 CRUD(增減維度)、招牌 tier 門檻調整、檢舉佇列覆核 / 手動 DQ、濫用偵測(同對反覆互評、異常差評爆量)。
