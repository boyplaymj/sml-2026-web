# 📒 記帳 (Ledger)

> **狀態**：升格橋強度定案(**使用者自控 + 接收方治理**，gameboy 2026-07-16)🔶 其餘 Claude 建議待微調
> 一句話：**記帳是 hook + 付費旗艦 + 成長引擎——低摩擦地記、深度復盤、無痛升成可信對局。**

三個身分集於一身：**hook**(500 人實證第一名功能)、**訂閱旗艦**(進階分析)、**成長引擎**(solo 自動長成可信對局 + 病毒拉新)。建立在現行 app 已有的 `/ledger`(日期/盈虧/狀態/照片/勝率統計)**之上深挖**。

---

## 1. 兩類分區(接 §3.1)

記帳頁 = **統一的「我的錢」**：私有 solo 紀錄 + 我參與的可信對局(唯讀鏡像、掛可信標)。solo `PK=USER#<uid>`、即時私有、不需確認。

## 2. 低摩擦捕捉(hook 命根，摩擦趨近零)

- **一步記帳**：打完一場，一個數字「今天 ___」按下即可;地點/牌搭/底台/照片/備註全**選填**。
- **智慧預填**：剛在某館 geofence 打卡、或剛結束排定團局 → 自動帶日期/場地/同桌。
- **快速再記**：沿用上次場地底台;下拉刷新(已有)。
- 🔶 **拍計分表照片自動帶金額(OCR)**：好用但吃 LLM 成本 → **列 v2**，MVP 先手動。

## 3. 個人財務體驗(免費/付費界線，接 business §2.5)

- **免費基礎**：損益總覽(累計輸贏/勝率/場次，已有)、近 N 筆、月度趨勢圖。
- **訂閱進階(旗艦鉤子)**：完整歷史 + 匯出;**多維盈虧拆解**——按場地/牌搭/底台/星期時段(「A 館勝率 60%、B 館 30%」「這底台長期輸」「週五晚場最賺」)。**自我復盤洞察**是重度玩家真正願付費的東西。
- 加分(責任遊戲好觀感)：**止損/預算提醒**、目標設定。

## 4. 無痛升 L1 + 病毒迴圈 ⭐(成長引擎)

- **升格機制**：solo 那筆若標了另三人且他們在 app → 一鍵**「邀請確認」**;缺的三家淨額提示「邀他們各自填」，**自動驗加總=0** 即升格(接 §3.2/§3.3)。
- **反向鏡像拉新**：有人確認一場 → 自動幫其餘三人**鏡像生成一筆帳**;被標到的**非 app 用戶**收到「你的牌搭幫你記了一筆，來確認/也自己記」→ **記帳把新用戶拉進來**(直攻媒合冷啟動空池)。
- **升格強度 = 使用者自控**(拆兩件事，關鍵)：
  - 🎚️ **「提示我自己」的強度**：使用者自選 `積極 / 標準(預設) / 安靜`，只影響**自己**看到多少升級入口。
  - 🛡️ **「主動通知別人/非用戶」**：**由接收方偏好治理，不由記帳者的積極度決定**(否則被標 = 被騷擾)。非用戶無設定 → 保守預設:**單次、去重、限流的溫和邀請，忽略就不再發**。
  - **預設「標準」**(多數人不改設定 → 這才是真決策)= 自己看到適度提示 + 對同桌 app 用戶送確認邀請 + 對非用戶送**一次**溫和邀請。積極=更頻繁自我提示;安靜=只留角落入口、完全不主動通知任何人。

## 5. 兩條護欄

- **金額永遠私密**：名片只顯示勝率/風評/場次，**絕不顯示輸贏金額**(§2.8 + 隱私)。錢只活在你自己的帳本。
- **金流紅線**：記的是使用者自填數字，現金 P2P offline，平台不經手不催收;`/overdraw`(透支)只當個人備註，平台**不強制不代收**。

## 6. 資料模型 🔶(擴充現有 `Ledger`)

```
Ledger { id, userId, date, amount(實際盈虧), status(結算狀態),
         photos[], note,
         venueId?, stake?(底台), coPlayers[]?,
         matchId?(升格後綁可信對局), source(solo | verified-mirror) }
UserPrefs { upgradeNudgeLevel(aggressive|standard|quiet),
            inviteOptOut(bool) }   // 接收方治理
```
API：`GET/POST /ledger`(已有)· `POST /ledger-upgrade`(發起升格,驗零和)· 進階分析端點(付費)。

## 💰 成本控管(遵循 tools/COST_CONTROL.md)

- **成本來源**：沿用現有 `ledger` 表(PAY_PER_REQUEST)+ 分析為讀時聚合/快取，量級小(< $1/月)。
- **OCR 拍照記帳(v2)若做** = LLM/視覺 → 走 Bedrock 無 key，並補齊帳本表/月封頂 cap/後台用量卡/kill switch 四件套。
- 非用戶邀請通知計入推播額度(§2.10)→ 限流節制。
