# 🧭 産品基石 (Foundation)

> 両雀 (Ryōjaku) 的世界觀、核心命題與**已鎖定的地基原則**。
> 這是所有模組設計的上位正典;新功能不得違反此處原則,若要改動需在此明確更新。
> 由 gameboy × Claude 規劃討論逐條敲定。最後更新:2026-07-15

---

## 0. 一句話

**両雀 = 一個封閉制的麻將牌咖社群:讓你能「安心約到牌品好、不放鴿子的牌搭子」,並把你的真實戰績,經營成一個看得到的角色與招牌。**

- **Hook(把人拉回來的日常)= 記帳。** 500 活躍用戶的資料實證:主力使用功能就是記帳。
- **護城河 = 信任。** 麻將揪團真正的痛點是放鴿子、牌品差、金錢糾紛、安全 → 封閉制 + 信譽是解法,不是裝高級。
- **成長引擎 = 把記帳升級成「可信對局」,再扇出到角色/場/天梯/RPG。**

---

## 1. 世界模型:人 · 場 · 局

| 實體 | 是什麼 | 它的「名片/招牌」 | 誰在讀 |
|---|---|---|---|
| **人(角色)** | 你這個玩家 | 等級 · 評價 · 外觀 · 稱號 | 想找咖的人 |
| **場(地標)** | 你開/經營的桌(持久) | 場評 · 狀況 · 熱度 · 特色 · 常客 | 想找場的人 |
| **局(對局)** | 實際打的每一場牌 | —(資料源) | 系統 |

**兩條經營線(人、場)各自獨立,但都靠同一份「可信對局資料」餵養。** 好場吸引好人、好人填滿好場 → 迴圈。

## 2. 已鎖定的地基原則

1. **定位**:信任=護城河;記帳=hook。一切功能為「信任」或「留存」服務。
2. **三元世界**:人 · 場 · 局。角色與場都可「經營」。
3. **名片鐵律**:角色/場身上每一個「看得到」的屬性,都必須對「正在決定要不要跟你打牌的人」有意義。純為升級而升級 = 雜訊,砍。
4. **外觀非照片**:用遊戲角色外觀,不用真實照片 → 保護隱私/安全、降低門檻、可養收集與變現。LINE(若接)只提供身分+通知,不拿其頭像當顯示。
5. **場=持久可經營**:不是一次性的局,而是會累積評價與常客的地標。個人場(社群場)與館家場(認證場)是**同一種實體、不同層級** → 統一「麻將館專區」,不做兩套。
6. **審核透明化**:主揪一定能逐一審核,可預設全過或熟客免審;但**審核制度、平均回應時間、通過率一律公開**當信譽訊號。申請人**絕不卡住**(逾時 fallback、可多申請先過先得、溫柔未錄取+推薦替代)。
7. **隱私=漸進式揭露**:地圖只給模糊範圍→場詳情給商圈地標→**核准後才給精確地址**。模糊點固定(防三角定位)、真實座標伺服器端分權(未授權者根本收不到)。**信譽名片=地址門禁**。
8. 🔒 **金流紅線**:App = 記帳 + 信任層,**絕不碰金流**(不代收代付、不抽賭金、不做賭資錢包)。營收只來自**服務費**:訂閱會員 / 外觀販售 / 館家合作。措辭一律用「計分/記帳/戰績」,現金 peer-to-peer,平台不定義金額幣值。**(上線前需台灣律師覆核灰色地帶)**
9. **地圖成本地基**:地圖**要有真實路網**(開車/騎車者靠路認方向,非人人搭捷運),但**絕不用「即時圖資 API」**(每次平移/縮放都計費=狂滑線性燒錢)。改走**自託管向量圖磚**:把台灣/大台北打包成一個 `.pmtiles`(含完整 OSM 路網)放自家 S3+CDN,MapLibre 讀靜態檔渲染 → 狂滑=讀快取靜態檔,**成本近乎固定**。用 MapLibre 自訂樣式 → **真實路 + 遊戲風美術兩全**。搭配:清單優先於地圖、場圖釘仍是模糊圈(隱私)、精確導航深連結外部地圖(Google/Apple)、geocode 只做一次。(懶得自託管的替代:MapTiler/Stadia 等**固定費率**供應商 + CloudFront 快取,非每磚計費。)詳見 [成本控管](cost)。
10. **通知分層**:App 內 / Web Push(免費)放日常;**LINE 可操作 Flex 通知**放關鍵事件(成局/缺腳快報/審核結果),計入月額度需節制。

## 3. 對局結算引擎 (Keystone)

一場牌打完 → 結果如何變成「各方認可」的可信紀錄,是整個系統的引擎室。

### 3.1 兩類分區(記帳的兩種現實)

記帳有兩種截然不同的現實,是**兩類東西**,不是同一光譜的強弱,必須**分區管理**:

| 類別 | 是什麼 | 確認 | 可信度 | 資料分區 | 下游 |
|---|---|---|---|---|---|
| **個人帳本 (Solo)** | 一個人自己記自己的帳(跟非 app 朋友打、線下臨時局、純追蹤輸贏/財務) | **不需要**,即時生效 | 私人、不可信 | `PK=USER#<uid>`(單人 owner、私有) | 只餵「我的財務體驗」 |
| **可信對局 (Verified)** | 四人零和 + 一人記 + 其餘確認 | **需要**(見 3.3 門檻) | 各方認可、可信 | `PK=MATCH#<id>`(多人共有、確認狀態機) | 成立才扇出下游(見 3.4) |

- **介面分區**:兩個清楚的 tab——「我的帳本」vs「可信對局」,不混在同一條時間軸,避免搞不清哪筆是認可的、哪筆只是自己記的。
- **個人帳本區**同時鏡像顯示「我參與的可信對局」(唯讀),讓使用者有**一個地方看到自己所有的錢**。

### 3.2 升格橋(別做成兩道牆)

魔力:**把「一個人的記帳」變「四個人的可信帳本」**。

- 帳本裡某筆 solo,若另三人也在 app → 一鍵「**發起確認**」→ 達門檻後**升格**成可信對局。
- 任何可信對局成立後,**自動鏡像一份唯讀**回四人各自的個人帳本。
- 另三人不在 app 的純 solo → 永遠留在個人帳本,這是預期內的常態,OK。

### 3.3 確認機制:三態 + 同意數門檻(治耍賴)

麻將一場是**四人零和**(淨額加總=0)→ 天然完整性檢查 + 天然確認單位。一人輸入四人淨額(自動驗加總=0)→ 其餘三人各收確認卡。

**確認人三態,以「主動秒點」為主**:
- **未回應** → **24h 逾時自動視為同意**。(純擺爛者擋不住紀錄成立。)
- **主動同意** → 計入同意。
- **主動不同意/異議** → 進下方門檻表。

**同意數門檻表**(最終同意數,含逾時自動同意):

| 最終同意數 | 結果 | 落點 |
|---|---|---|
| **4** | ✅ 可信對局(乾淨) | 可信對局區 + 鏡像回各人帳本 |
| **3**(1人主動異議) | ✅ 可信對局,**掛「⚠️1人異議」標示** | 可信對局區(標示)+ 鏡像。下游依 3.4 部分擱置 |
| **2**(2人異議) | ❌ 不成立 | 只落各自個人帳本 |
| **1**(只有自己) | ❌ 不成立 | 只落個人帳本 |

一句話:**≥3 同意即成立(不滿 4 就標示);<3 一律退回個人帳本。**(此規則**取代**舊的「異議即凍結」。)

### 3.4 一事件扇出六系統 + 「帶異議」的部分擱置(方案B)

可信對局成立 → 自動觸發 `記帳(四人) · 評價窗口 · 名片統計 · 天梯積分 · RPG成長 · 場評`,六模組訂閱這一個上游事件。

**但「3/4 帶異議」的紀錄,不能讓 3 人在 1 人不認下硬壓其信譽**,故分兩批:
- **立即成立(帳與自身)**:記帳(四人版) + 個人帳本鏡像 + RPG成長(自身養成)。
- **擱置到異議解除**:凡「對外信譽/競技/場評」訊號——**天梯積分 · 信譽評價(牌品/準時) · 場評 · 名片對外統計**——一律先不餵。
- **解除路徑**:異議者撤銷異議、或修正數字重新確認變 4/4 → **補餵(追認)**。若長期不解除 → 信譽類**持續擱置**(保護誠實異議者),帳與 ⚠️ 標示永久保留。(「長期不解除是否設追認時效」見 §6。)

**其餘信任機制**:零和檢查擋輸入錯誤;記帳者具名可課責;顆粒度 L1(場結算·四人確認,⭐預設)/ L2(逐局計分,餵 RPG 打法風格+天梯精算)皆在可信對局區內。

## 4. 核心流程(端到端)

```
找場(定位·依距離排招募中的場)
  → 讀 場評 + 主揪名片
  → 一鍵申請加入(名片門檻自動篩 / 或主揪審核)
  → 核准 → 地址解鎖 + 進專屬聊天室(成局卡:地址·導航·時間·底台·各人名片)
  → 打牌
  → 對局結算(一人記·四人確認)= 可信對局
  → 扇出:記帳·評價·名片·天梯·RPG·場評
```
輔助:⚡快速湊咖(想打→媒合池)、🔔缺腳快報(差1腳→推附近想打的人一鍵補位)——救 500 人小池子流動性。
