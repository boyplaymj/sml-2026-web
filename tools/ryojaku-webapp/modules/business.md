# 💼 商業模式 (Business Model)

> **狀態**：三條營收線由 §2.8 紅線鎖定；規劃 + 成長框架已由 gameboy 拍板(2026-07-16)🔶 細節待微調
> 一句話：**不碰金流，營收只來自服務——訂閱 / 外觀 / 館家；規模綁在「賽事+直播」漏斗上，先修三瓶頸再灌量。**

---

## 0. 紅線(繼承 §2.8，不可破)

不碰金流(不代收代付/不抽賭金/不做賭資錢包)。措辭一律「計分/記帳/戰績」，現金 peer-to-peer、平台不定義金額幣值。**整套上線前需台灣律師覆核**(商業模式是全 app 最貼近賭博灰線的一塊)。

---

## 1. 成長框架：500 是被壓抑的地板，不是天花板

現有 ~500 活躍**不是市場上限**，是三個瓶頸壓出來的低點：

| 瓶頸 | 現況 | 解 |
|---|---|---|
| **推廣未開** | 手握最強漏斗(**賽事 + 直播**)尚未導入 | 產品補好後開閘 |
| **產品漏水桶** ⚠️最致命 | 連「忘記密碼」都無法找回 → 帳號健康缺失 | **變現前提**:先修帳號找回/核心可靠 |
| **媒合冷啟動空池** | =推廣未開的果，池薄湊不到人 | 用 #1 資產播種(見下) |

**排序鐵律**：**先補水桶(帳號健康)→ 用直播播種流動性 → 再開閘灌量。** 首播注意力是最稀缺資產，觀眾只有一次第一印象；拿賽事+直播去灌一個會漏的桶 = 燒王牌。

**直播 = 流動性種子**：賽事/直播可**直接注入媒合池**(現場開放報名抽台、串進缺腳快報)→ 冷啟動一次破。

**含意**：C 端**不是零錢**，規模綁直播漏斗 → **C 端 + 館家 B2B 雙引擎**(非「只能靠 B2B」)。

## 2. 三條營收線 + 免費/付費界線

| 線 | 定位 | 付費得到 | 免費線 |
|---|---|---|---|
| **① 訂閱會員** | 留存 + 回收成本 | 進階記帳分析(圖表/匯出/長期歷史)、開團配額、更廣媒合半徑、月贈外觀點、天梯進階視圖 | **完整信任社交核心** + 基礎記帳(近 N 筆) |
| **② 外觀販售** | 高毛利、可規模化 | 角色組件/邊框/稱號/名片主題/場招牌(§2.4 非照片可變現，素材 Bedrock 生成) | 基礎外觀 + 通行證免費軌 |
| **③ 館家合作** | B2B、高客單支柱 | 認證徽章 + 置頂曝光 + 客流分析(缺人時段)+ 招募工具 + 常客 CRM-lite；月訂 + 抽**服務費**(非賭金) | — |

- **① 記帳為何能收費**：記帳是 hook，重度用戶最願為 hook 的**深度**付錢。
- **② 外觀模型**：**賽季通行證(battle pass)+ 直購**。通行證用**真實對局進度**解鎖(綁 §2 養成綁真實對局非掛機)→ 同時促留存與變現。
- **③ 館家**：客單是 C 端數十倍、需 BD，中期線;但 C 端規模隨直播起來後兩引擎並進。

### 2.5 付費解鎖功能目錄(基本免費 · 訂閱解更多，可持續擴充)

> 鐵則：付費解鎖的是**表現力 + 便利 + 曝光**，**絕不是安全 / 可信度 / 公平**。

| 對象 | 免費(核心不缺) | 訂閱解鎖「更多功能」 |
|---|---|---|
| **玩家** | 找場 / 揪團 / 可信對局 / 信譽 / 爭議處理 / 檢舉 / 基礎記帳 / 標準地圖圖釘 | 進階記帳分析、角色進階外觀 / 稱號、更廣媒合半徑、更多同時揪團、天梯進階視圖 |
| **場主** | 標準圖釘(模糊圈、找得到)、基礎場資料 | ⭐ **地圖自訂圖標**、場招牌裝飾 / 主題、多場管理、常客名單、開場曝光配額 |
| **館家(B2B)** | — | 認證徽章(**系統核發、不可購買**)、置頂曝光、客流分析、CRM |

**目錄護欄**(套用整張表)：① 自訂圖標騎在**模糊圈**上、不露精確址(§2.7);② 付費裝飾**不得假冒認證徽章**(認證=打真牌賺來、不可買);③ 免費場**永遠找得到**(不 pay-to-win)。

---

## 3. 三條不可破的護欄

1. **信任/安全永不付費牆**：信譽、爭議處理、檢舉、地址門禁、反分身審核 → **全免費核心**。收費安全 = 自毀護城河。
2. **絕不 pay-to-win**：不賣天梯分/勝率;外觀純裝飾，**不影響對局或媒合公平**。
3. **🚫 不做轉蛋/gacha**：機率型付費在走賭博灰線的 app 是地雷(《機率型商品》規範 + 賭博觀感)→ 只做直購/通行證。金流走 IAP 或綠界訂閱=「服務費」合規。

---

## 4. 上線排序(roadmap)

- **P0｜產品健康**(帳號找回、核心可靠)= **一切變現的前提**，先於任何收費。
- **P1｜C 端**(訂閱 + 外觀通行證)隨**直播漏斗**同步放量。
- **P2｜B2B 館家**(認證/客流/服務費)BD 開發。
