# ⛩️ 甜甜神社 — 參拜作法（礼儀・儀式）spec

> **神社的核心玩法 = 把參拜作法做對 → 増運;失礼/亂序 → 扣運。** 運氣全隱藏黑箱(玩家只得神職的定性回饋、看不到數字)。
> 這份把散落在對話的礼儀規則整合成正典:入場礼 → 手水五步 → 参拜 → 其他 → 退場礼,以及違反的罰則與 visit 狀態機。
> 定稿:2026-07-19。影響 S3(導覽骨架要納入 visit 狀態機,不再只是換圖)。

---

## 1. visit 狀態機(核心)

每位玩家一趟參拜 = 一筆 **visit 狀態**(存 `fortune.visit` 或帶 TTL 的 visit 記錄,per discordId 一筆):
```jsonc
{
  "visitId": "...",         // 本趟識別
  "enteredAt": 1721000000,
  "bowedIn": false,          // 鳥居で一礼したか
  "temizuState": [],         // 手水已按的步驟序(驗順序用)
  "temizuDone": false,       // 手水を正しく終えたか(當日/趟有效,見待拍板)
  "worshipped": false,       // 本殿で参拜したか
  "bowedOut": false,         // 退場で一礼したか
  "fail": []                 // 失礼記録(['temizu_order','no_temizu_worship',...]) → 結算時扣運
}
```
- **lazy 結算**:失礼不即時扣,累積在 `fail`;於**退場**或**下次進場發現上趟未乾淨關閉**時,一次把 `fail` 折成負向 buff 寫入(對齊神社 lazy compute)。
- 失礼形式 = append 負向「失礼の咎」buff(綜合運 −X、N 天)到 `fortune.buffs`,黑箱不顯數字。

## 2. 正しい作法(順序)→ 増運

```
1. 鳥居で一礼      [🙇 一礼して入る]         → bowedIn
2. 手水舍で清める  5 步互動(§3)            → temizuDone
3. 本殿で参拜      (要 temizuDone,否則失礼) → 加持(おみくじ/buff)
4. 其他設施        授與所/御朱印/御祈禱/市集/奧社
5. 鳥居で一礼      [🙇 一礼して退く]         → bowedOut,visit 乾淨關閉
```

## 3. 手水の作法（5 步互動；順序錯 → 扣運）

手水舍面板:描述區放作法本文,下方 **5 顆按鈕(位置每次打散)**,玩家須照正解序按。

| 正解序 | 按鈕(label) | 動作說明 |
|---|---|---|
| 1 | 🥢 左手を清める | 右手で柄杓を持ち水を汲み、左手を清める |
| 2 | ✋ 右手を清める | 柄杓を左手に持ち替え、右手を清める |
| 3 | 💧 口をすすぐ | 柄杓を右手に戻し、左手の掌に水を受けて口をすすぐ(**柄杓に直接口をつけない**) |
| 4 | 🖐 再び左手を清める | 残りの水で左手をもう一度清める |
| 5 | 🥄 柄杓を清める | 柄杓を垂直に立て、残った水で柄を流し清め、元へ戻す |

**互動邏輯**
- customId `shrtemizu` + `[stepId]`;handler 讀 `visit.temizuState`,驗 `stepId === 期待的下一步`。
- **按對** → push 進 `temizuState`、面板更新(已完成步驟顯示 ✓)。
- **按錯順序** → `fail.push('temizu_order')`(一次/趟,不重複累加)→ 失礼扣運 + `temizuState` 清空(**手水重來**)。神職斥責文字。
- **5 步全對** → `temizuDone=true` → 微加持(守禮の心)→ 提示可往本殿参拜。
- 按鈕位置**打散**(避免純肌肉記憶;作法本文有寫,屬「照做能不能專心」而非記憶測驗)。

## 4. 罰則 / 獎勵（config 草案，可調）

| 行為 | 效果 |
|---|---|
| 入場一礼 | **門**(建議必按才入境;強制守禮=正向體驗)。或極小加持 |
| 手水 正しく完了 | +微加持(守禮) |
| 手水 順序錯 | 失礼:綜合 −5 / 3 天(一趟一次)+ 手水重來 |
| 手水せず参拜 | 失礼 + 参拜**不給加持**(或給負) |
| 正しく参拜(手水済) | 加持(抽籤/buff,黑箱) |
| 退場一礼せず放置 | 失礼(lazy,下次進場補扣) |
| 順序全対で一巡 | 綜合的な守禮ボーナス(可選) |

> 全部走既有 buff 機制(正/負 buff,帶 expireAt),黑箱。config 可調每項量與天數。

## 5. 決策(2026-07-19 定案，照建議；全 config 可調)

1. ✅ **手水前禁止範圍**:僅「参拜系(本殿/御祈禱)」需先手水;**純購物(授與所/御朱印/市集)豁免**(買東西不必淨身)。
2. ✅ **手水時效**:**當日有效**(對齊 DESIGN 手水=每日 gate,同日再訪免重洗;`fortune.lastTemizuDate`)。
3. ✅ **手水按鈕**:**位置每次打散**。
4. ✅ **入場一礼**:**硬門**——不按 `[🙇 一礼して入る]` 不得入境(強制守禮=正向體驗)。真正會忘的是退場礼 → 那個走 lazy 補扣。
5. ✅ **數值草案採用**(config 可調):守禮=微加持;失礼(手水亂序/沒手水就參拜/退場沒鞠躬)=綜合 −5 / 3 天,一趟一次不重複累加。守禮=微正 buff。
