# ⛩️ 甜甜神社 — 參拜作法（礼儀・儀式）spec

> **神社的核心玩法 = 把參拜作法做對 → 増運;失礼/亂序 → 扣運。** 運氣全隱藏黑箱(玩家只得神職的定性回饋、看不到數字)。
> 這份把散落在對話的礼儀規則整合成正典:入場礼 → 手水五步 → 参拜 → 其他 → 退場礼,以及違反的罰則與 visit 狀態機。
> 定稿:2026-07-19。影響 S3(導覽骨架要納入 visit 狀態機,不再只是換圖)。

---

## 1. visit 狀態機(核心)

每位玩家一趟參拜 = 一筆 **visit 狀態**(存 `fortune.visit` 或帶 TTL 的 visit 記錄,per discordId 一筆):
```jsonc
{
  "visitId": "...",         // 本趟識別
  "enteredAt": 1721000000,
  "bowedIn": false,          // 鳥居で一礼したか
  "temizuState": [],         // 手水「進行中」暫存:本趟已按的步驟序(僅未完成流程用,非每日成效來源)
  "worshipped": false,       // 本殿で参拜したか
  "bowedOut": false,         // 退場で一礼したか
  "fail": []                 // 失礼記録(['temizu_order','no_temizu_worship',...]) → 結算時扣運
}
```
- **lazy 結算**:失礼不即時扣,累積在 `fail`;於**退場**或**下次進場發現上趟未乾淨關閉**時,一次把 `fail` 折成負向 buff 寫入(對齊神社 lazy compute)。
- 失礼形式 = append 負向「失礼の咎」buff(綜合運 −X、N 天)到 `fortune.buffs`,黑箱不顯數字。

## 2. 正しい作法(順序)→ 増運

```
1. 鳥居で一礼      [🙇 一礼して入る]         → bowedIn
2. 手水舍で清める  5 步互動(§3)            → 判定當日 temizuMult(§4)
3. 本殿で参拜      → 加持(おみくじ/buff) × temizuMult(未手水僅 0.2,見 §4)
4. 其他設施        授與所/御朱印/御祈禱/市集/奧社
5. 鳥居で一礼      [🙇 一礼して退く]         → bowedOut,visit 乾淨關閉
```

## 3. 手水の作法（5 步互動；順序錯 → 扣運）

手水舍面板:描述區放作法本文,下方 **5 顆按鈕(位置每次打散)**,玩家須照正解序按。

| 正解序 | 按鈕(label) | 動作說明 |
|---|---|---|
| 1 | 🥢 左手を清める | 右手で柄杓を持ち水を汲み、左手を清める |
| 2 | ✋ 右手を清める | 柄杓を左手に持ち替え、右手を清める |
| 3 | 💧 口をすすぐ | 柄杓を右手に戻し、左手の掌に水を受けて口をすすぐ(**柄杓に直接口をつけない**) |
| 4 | 🖐 再び左手を清める | 残りの水で左手をもう一度清める |
| 5 | 🥄 柄杓を清める | 柄杓を垂直に立て、残った水で柄を流し清め、元へ戻す |

**互動邏輯（v0.3 定案：每日第一次定生死、多做不算、不能重做救回、全程無提示）**
- customId `shrtemizu` + `[stepId]`;handler 把玩家按的 stepId push 進 `visit.temizuState`。
- **全程無提示對錯**:按任何一顆只顯「已進行」,面板**不透露**順序正確與否(運氣黑箱)。
- **5 顆全按完** → 一次判定 `temizuState === 正解序 [1,2,3,4,5]`,寫當日結果(§4 temizuMult):
  - **全對** → `temizuMult=1.0`(守禮,全額運氣)。
  - **有錯** → `temizuMult=0.5` + **一次性失礼扣幸運值**(append 負 buff,黑箱)。
- **當日只認第一次**(`fortune.temizuDate===今日` 已定)→ 之後再來手水,**做幾次都不改結果**(多做不算、不能重做救回)。神職只給氛圍文字。
- 未按滿 5 顆(中途離開)= 手水未完成 = 當日仍視為未做(`temizuMult=0.2`),下次可重新開始(尚未定生死)。
- 按鈕位置**每次打散**(照做能不能專心,非記憶測驗)。

## 4. 手水乘數 temizuMult（運氣取得的效率係數）— 核心機制

**手水做對 → 全額運氣;沒做/做錯 → 運氣提升打折。** temizuMult 套用到**所有運氣「提升」的取得**。

| 當日手水狀態 | temizuMult | 效果 |
|---|---|---|
| ❌ 未做手水 | **0.2** | 運勢提升 **80% off** |
| ⚠️ 有做但順序錯 | **0.5** | **50% off** + 一次性失礼扣幸運 |
| ✅ 五步順序正確 | **1.0** | 全額 |

- **套用點**:御神籤首抽 buff、御守 buff、御朱印 buff、本殿参拜加持 —— 取得時 `delta *= temizuMult`(正向 delta 才乘)。
- **只折正向**:負面(凶籤扣運 / 失礼 / pendingKyo / 穢れ)**一律不打折**(否則沒手水反而減輕懲罰,邏輯反了)。
- **當日有效、隔日歸零**:`fortune.temizuDate`(台北日)≠今日 → 視為未做=0.2 → **每天都要重新手水**才有全額。
- 黑箱:玩家看不到 mult 數字,只由神職氛圍文字暗示。存 `fortune.temizuMult` + `fortune.temizuDate`。

### 4.1 其他礼（與 temizuMult 並存）
| 行為 | 效果 |
|---|---|
| 入場一礼 | **硬門**——不按 `[🙇 一礼して入る]` 不得入境 |
| 退場一礼せず放置 | 失礼(lazy,下次進場補扣負 buff) |

> 全部走既有 buff 機制(正/負 buff 帶 expireAt),黑箱。config 可調每項量與天數、三檔 mult 值。

## 5. 決策(2026-07-19 定案，照建議；全 config 可調)

1. ✅ **手水＝軟折扣非硬擋**(v0.3 改):**不擋任何設施**,一律吃 temizuMult(§4)。沒手水去參拜/買御守/抽籤/蓋御朱印 → 運氣提升只剩 20%(照付全額牙齒、只拿 20% buff)。原「參拜系需先手水、購物豁免」的硬 gate **取消**。
2. ✅ **手水時效**:**當日有效**(同日再訪不重判、多做不算;唯一欄位＝**`fortune.temizuDate`(台北日) + `fortune.temizuMult`**,隔日歸零視為未做=0.2。**不再用 `lastTemizuDate`/`temizuDone` 舊語彙**;visit.temizuState 僅未完成流程暫存)。
3. ✅ **手水按鈕**:**位置每次打散**。
4. ✅ **入場一礼**:**硬門**——不按 `[🙇 一礼して入る]` 不得入境(強制守禮=正向體驗)。真正會忘的是退場礼 → 那個走 lazy 補扣。
5. ✅ **數值草案採用**(config 可調):守禮=微加持;失礼(手水亂序/沒手水就參拜/退場沒鞠躬)=綜合 −5 / 3 天,一趟一次不重複累加。守禮=微正 buff。
