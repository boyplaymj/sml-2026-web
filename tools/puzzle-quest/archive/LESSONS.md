> 累積歷次封測玩下來、玩家/主持人抓到的問題。**出新題前先掃一遍這張總表，逐條自檢**，再對 `../DESIGN_DIRECTION.md` §5 checklist。
> 新題玩完，把新踩到的雷補進 §1、把該題結論補進 §2 與 `README.md`。

## §1 通用踩雷清單（每題都要自檢）

### 難度／結構
- [ ] **答案太簡單**（CASE-09）：早階段（S1/S2）就湊得齊正解 → 破。修法見 DESIGN_DIRECTION §1「拱心石只在 S4」。
- [ ] **提示曝光關鍵字**（CASE-09）：hint／nudge 直接講出 core 關鍵名詞（手法/動機/對象）→ 買一個提示就破台。hint 只能指路（看哪張、哪兩張對照、時間兜不兜得上），禁出現 core 名詞。
- [ ] 沒有「站得住腳的錯誤答案」把人卡住往下挖。
- [ ] **素材出現階段 vs 提示階段要一致**（CASE-10）：某份關鍵文件（如驗屍/相驗報告）在早階段（S1）就被提到／暗示存在，卻要到 S4 才真的出現 → 玩家一開始就想看、卻卡好幾階段看不到，很違和。規則：一份素材「被提及的時機」要對齊「它出現的時機」——拱心石可鎖 S4，但那就**別在早階段提它的存在**；反過來若早階段要營造「有這份報告」的存在感，該階段至少給部分/遮蔽版。做每張圖前，把它「第一次被提到的階段」與「appear 階段」寫在一起自檢。

### 電話 AI 角色（海龜湯 NPC）
- [x] **連問數次無新線索要有保底引導**（CASE-10）✅**機制已做**（commit cdc80b8）：`PuzzlePhoneNPC` 連續 ≥3 次沒問到新線索、到第 4 次起自動追加溫和方向；靠 NPC 自標 `[[NEW]]/[[SAME]]`（自動剝除）判定。**出新題只需在 json 補 `npc.fallbackNudges`**（2 條、不含 core/keystone，見 DESIGN_DIRECTION §3）。

### 階段推進機制（主持人 UX）
- [ ] **「推進到下一階段」門檻難拿捏**（CASE-10 主持人回饋）：主持人不易判斷何時該推（玩家還在挖 vs 已卡住）。可考慮：面板顯示該階段停留時間／作答次數當參考，或給「軟提示」節奏建議。
- [ ] **推進是「重貼新 embed」而非「原地改」——與預期落差**（CASE-10）：目前 `advanceActiveCase` 在頻道 **repost 一則新 panel embed**；使用者原以為是「偷偷 edit 原本那則 embed」讓劇情無聲演進、不洗版。→ 待評估改成 `messageEdit`（用 round 已存的 `messageId`/`messageUrl` 就地更新）而非 send 新訊息。列為機制待改，不只文案。

### 素材文字／截圖（做圖時最容易錯）
- [ ] **手機截圖時間不正確**（CASE-09）：LINE／簡訊截圖的時間戳、與案件鎖定時間軸（§7）對不上，或幾張截圖之間先後矛盾。做圖前先把每張截圖該顯示的時間寫死、彼此對齊。
- [ ] **簡體字混入顯示文字**（CASE-09）：台灣情境的截圖/文件/新聞出現簡體字（「坠、盖、装、发、们、后…」）→ 立刻出戲。**所有會被玩家看到的圖上文字一律繁體**。
      - ⚠️ 例外：`solution.core.any` 的比對詞庫**可以**留幾個簡體變體（用來接玩家打簡體的作答），那是後端比對、不顯示，OK；但**顯示文案零簡體**。
- [ ] emoji 在截圖會變豆腐框 → 改文字或 inline SVG（見 HANDOFF §4 製圖管線）。
- [ ] 錯字、標點全形/半形混用、公司/機關名前後不一致。
- [ ] **車輛場景一律左駕（台灣）＋多張要一致**（CASE-11）：Bedrock 生車內/車禍圖常隨機出成右駕（方向盤在右）。台灣是左駕（方向盤在左、靠右行駛）。生圖 prompt 明寫 "left-hand-drive, steering wheel on the LEFT"，且**同案多張車輛圖的駕駛座方向要彼此一致**（現場照 vs 駕駛座特寫別一左一右）；生完逐張確認方向盤在左。

### 邏輯／一致性
- [ ] 時間軸矛盾（歇業年份 vs 廢墟仍在、生年 vs 享壽…；puzzles.json 歷史上被玩家抓過多次）。
- [ ] 不在場證明／門禁／監視器的可得性彼此矛盾（CASE-09：監視錄影暫停但門禁刷卡仍記錄——這種「哪個有紀錄哪個沒有」要前後一致）。
- [ ] 生前傷／死後傷、死亡時間推定等鑑識細節，要跟 reveal 對得上。

### 分級／合規
- [ ] 全虛構，勿影射真實個案/公司/人物/判決（即使改編自真實案例，人名地名情節要虛構化）。
- [ ] reveal 收斂在「重症送醫」等級，避免露骨。

### 成本
- [ ] 燒 LLM（電話 AI）者沿用 `sweetbot-puzzle-ai-usage` 帳本＋月封頂＋後台卡＋kill switch（`tools/COST_CONTROL.md`）。
